# Phase 2.0: Contract-Awareness Module — Design Document

**Status**: Design Complete (Ready for Implementation)
**Date**: 2026-03-04
**Prerequisites**: Phase 0 (Contract Generation) & Phase 1 (Page Cache) Complete

## Executive Summary

Phase 2.0 introduces a **deterministic, runtime-safe Contract Awareness module** that loads and validates Phase 0 contract artifacts, binds Phase 1 page cache, and exposes a comprehensive **Strategy Context** for runtime execution, healing, and cross-repo tooling.

**Core Principles:**
- ✅ **Additive only** - Zero breaking changes to existing Phase 2.2 runtime
- ✅ **Read-only** for `.mcp-contract/*` and `.mcp-cache/*`
- ✅ **Write-only** for `runs/<runId>/artifacts/**`
- ✅ **Deterministic** - No network calls, no randomness, byte-identical outputs
- ✅ **Mode-aware** - COMPLIANCE (strict) vs BEST_EFFORT (permissive)
- ✅ **Exit code 3** for compliance violations in COMPLIANCE mode

**Key Changes from Phase 2.2:**

| Aspect | Phase 2.2 (Existing) | Phase 2.0 (NEW) |
|--------|----------------------|-----------------|
| Location | `runtime/contract-*.ts` | `contract-awareness/*` |
| Validation | Basic snapshot reading | Full AJV schema validation |
| Fingerprint | Not verified | SHA256 integrity check |
| Cache binding | Not enforced | Hash matching enforced |
| Strategy context | None | Comprehensive selector/policy context |
| Error taxonomy | String warnings | Structured issues with severity |
| Exit code 3 | No | Yes (COMPLIANCE mode) |

- Contract files

### 2.3 Validation (`validator.ts`)

**Schema Validation (AJV):**

```typescript
function validateContractSchemas(loaded: LoadedContractBundle): ValidationIssue[] {
  const issues: ValidationIssue[] = [];
  const ajv = new Ajv({ allErrors: true, strict: true });
  addFormats(ajv);

  // Load schemas from contract-awareness/schemas/ (copied at build)
  const schemas = loadSchemas();

  for (const [file, schema] of Object.entries(schemas)) {
    const data = loaded.files[file];
    if (!data) continue;

    const validate = ajv.compile(schema);
    const valid = validate(data);

    if (!valid) {
      issues.push({
        code: "CA_SCHEMA_INVALID",
        category: "schema",
        severity: "ERROR",  // Catastrophic even in BEST_EFFORT
        message: `Schema validation failed for ${file}`,
        details: { errors: validate.errors, file }
      });
    }
  }

  return issues;
}
```

**Fingerprint Verification (mode-aware):**

```typescript
function verifyFingerprint(
  contractDir: string,
  mode: "COMPLIANCE" | "BEST_EFFORT"
): FingerprintResult {
  const hashPath = join(contractDir, "automation-contract.hash");

  if (!existsSync(hashPath)) {
    return {
      match: false,
      stored: null,
      computed: null,
      issues: [{
        code: "CA_HASH_MISSING",
        category: "fingerprint",
        severity: "ERROR",
        message: "automation-contract.hash not found"
      }]
    };
  }

  const storedHash = readFileSync(hashPath, "utf-8").trim();

  // Use matching mode for fingerprint computation
  const fingerprintMode = mode === "COMPLIANCE" ? "strict" : "best_effort";
  const computed = computeContractFingerprint(contractDir, fingerprintMode);

  if (!computed.ok) {
    return {
      match: false,
      stored: storedHash,
      computed: null,
      issues: [{
        code: "CA_HASH_COMPUTE_FAILED",
        category: "fingerprint",
        severity: "ERROR",
        message: computed.error
      }]
    };
  }

  const match = computed.fingerprint === storedHash;

  return {
    match,
    stored: storedHash,
    computed: computed.fingerprint,
    files: computed.files,
    issues: match ? [] : [{
      code: "CA_HASH_MISMATCH",
      category: "fingerprint",
      severity: "ERROR",
      message: `Fingerprint mismatch. Expected: ${storedHash}, Got: ${computed.fingerprint}`,
      details: { stored: storedHash, computed: computed.fingerprint }
    }]
  };
}
```

**Hash Binding (always non-fatal):**

```typescript
function bindCacheToContract(
  cacheDir: string,
  isLegacy: boolean,
  contractHash: string
): CacheBindingResult {
  const issues: ValidationIssue[] = [];

  // Read cache metadata
  let cacheHash: string | null = null;
  let pagesCount = 0;

  if (isLegacy) {
    // Legacy: read index.json
    const indexPath = join(cacheDir, "index.json");
    if (existsSync(indexPath)) {
      const index = JSON.parse(readFileSync(indexPath, "utf-8"));
      cacheHash = index.contractSha256 || null;
      pagesCount = index.pages?.length || 0;
    }
  } else {
    // Canonical: read v1/meta.json
    const metaPath = join(cacheDir, "meta.json");
    if (existsSync(metaPath)) {
      const meta = JSON.parse(readFileSync(metaPath, "utf-8"));
      cacheHash = meta.contract_hash || null;
      pagesCount = meta.pages_count || 0;
    }
  }

  const match = cacheHash === contractHash;

  if (!match) {
    issues.push({
      code: "CA_CACHE_HASH_MISMATCH",
      category: "cache",
      severity: "WARN",  // Cache issues are ALWAYS warnings
      message: `Cache hash mismatch. Expected: ${contractHash}, Got: ${cacheHash || "null"}. Cache will be ignored.`,
      details: { contractHash, cacheHash }
    });
  }

  return {
    valid: match,
    cacheHash,
    contractHash,
    pagesCount: match ? pagesCount : 0,  // Zero if invalid
    issues
  };
}
```

### 2.5 Strategy Context Builder (`strategy.ts`)

```typescript
function buildStrategyContext(params: {
  loaded: LoadedContractBundle;
  cache: CacheBindingResult;
}): ContractStrategyContext {
  const { loaded, cache } = params;

  // Extract from contract files
  const automationContract = loaded.files["automation-contract.json"];
  const frameworkPattern = loaded.files["framework-pattern.json"];
  const selectorStrategy = loaded.files["selector-strategy.json"];
  const assertionStyle = loaded.files["assertion-style.json"];
  const pageKeyPolicy = loaded.files["page-key-policy.json"];

  return {
    framework: frameworkPattern.framework,
    architecture: frameworkPattern.style,
    primaryStyle: automationContract.primaryStyle,

    // Selector preferences with deterministic ranking order
    selectorPreference: selectorStrategy.preferenceOrder,
    selectorRanking: [
      "contract_strategy",      // 1. Contract locator strategy (primary)
      "semantic_cache",         // 2. Semantic page cache (high confidence)
      "last_known_good",        // 3. Historical fallback
      "deterministic_fallback", // 4. Guaranteed coverage
      "llm_advisory"            // 5. LLM-assisted (advisory only)
    ],

    // Risk rules with deterministic defaults
    riskRules: {
      allowXPath: selectorStrategy.riskRules?.allowXPath ?? false,
      cssLastResort: selectorStrategy.riskRules?.cssLastResort ?? true,
      requireStableIdsWhenAvailable: selectorStrategy.riskRules?.requireStableIdsWhenAvailable ?? true
    },

    // Page identity policy
    pageKeyPolicy: {
      patterns: pageKeyPolicy.patterns,
      collisionResolution: pageKeyPolicy.collisionResolution || "deterministic_suffix"
    },

    // Style-aware entrypoints
    entrypoints: automationContract.paths?.entrypoints || {},

    // Assertion style
    assertionStyle: assertionStyle.primary,
    assertionWrappers: assertionStyle.wrappers || [],

    // Cache binding status
    cache: {
      valid: cache.valid,
      bound: cache.valid,
      pagesLoaded: cache.pagesCount,
      directory: cache.valid ? params.loaded.cacheDir : null,
      isLegacy: params.loaded.cacheIsLegacy
    },

    // Contract metadata
    contract: {
      hash: loaded.contractHash,
      schemaVersion: automationContract.schema_version,
      directory: loaded.contractDir,
      isLegacy: loaded.isLegacy
    }
  };
}
```

**Atomic Write with Canonical JSON:**

```typescript
function writeRuntimeContractContext(params: {
  artifactsDir: string;
  loaded: LoadedContractBundle;
  validation: ContractValidationResult;
  cache: CacheBindingResult;
  strategy: ContractStrategyContext;
}): string {
  const { artifactsDir, loaded, validation, cache, strategy } = params;

  const output = {
    schema_version: "1.0.0",
    runId: basename(dirname(artifactsDir)),
    contract: {
      hash: loaded.contractHash,
      schema_version: strategy.contract.schemaVersion,
      framework: strategy.framework,
      directory: strategy.contract.directory,
      isLegacy: strategy.contract.isLegacy
    },
    cache: {
      valid: cache.valid,
      hash_match: cache.valid,
      pages_loaded: cache.pagesCount,
      directory: strategy.cache.directory,
      isLegacy: strategy.cache.isLegacy
    },
    strategy: canonicalStringify(strategy),  // Deterministic JSON
    issues: validation.issues.map(i => ({
      code: i.code,
      category: i.category,
      severity: i.severity,
      message: i.message
    }))
  };

  const outputPath = determineArtifactPath(artifactsDir);
  const tempPath = outputPath + ".tmp." + randomUUID();

  // Atomic write: temp → rename
  writeFileSync(tempPath, canonicalStringify(output), "utf-8");
  renameSync(tempPath, outputPath);

  return outputPath;
}
```

```
  ↓

  ↓

  ↓

  ↓
[12] Post-run artifacts (existing)
```

### 3.2 Critical Timing Invariants

**Invariant 1: Contract-awareness before test execution**
- Steps [3-7] run **BEFORE** Playwright spawns
- Exit code 3 happens at step [4] if COMPLIANCE mode and errors found
- No test output if contract validation fails

**Invariant 2: policy-decision.json before healing**
- Step [10] writes `policy-decision.json` **BEFORE** any healing attempts
- Healing engine (Phase 3+) reads this file to determine policy
- Contract-awareness does NOT write policy-decision.json

**Invariant 3: No process.exit() in library code**
- Phase 2.0 module returns structured results (never calls `process.exit()`)
- Runtime orchestrator decides when to exit
- Ensures audit trail and exit.json are written before exit

### 3.3 Phase 2.2 Wrapper Integration

**Modified `contract-plumbing.ts` (delegates to Phase 2.0):**

```typescript
import {
  loadContractBundle,
  validateContractBundle,
  bindCache,
  buildStrategyContext,
  writeRuntimeContractContext
} from "../contract-awareness/index.js";

export function applyRuntimeContractContextEnv(args: {
  artifactsDir: string
}): RuntimeContractContext {
  const mode = process.env.MINDTRACE_MODE === "strict" ? "COMPLIANCE" : "BEST_EFFORT";
  const repoRoot = findRepoRoot(args.artifactsDir);

  // Phase 2.0: Full contract loading + validation
  const loaded = loadContractBundle({ repoRoot, mode });
  const validation = validateContractBundle({ loaded, mode });

  // Check for ERROR issues (but don't exit here)
  const hasErrors = validation.issues.some(i => i.severity === "ERROR");

  // COMPLIANCE gate: Write validation artifact and return exit signal
  if (mode === "COMPLIANCE" && hasErrors) {
    // Write validation issues to artifact
    const validationPath = join(args.artifactsDir, "phase0", "validation.json");
    mkdirSync(dirname(validationPath), { recursive: true });
    writeFileSync(validationPath, canonicalStringify({
      valid: false,
      issues: validation.issues
    }), "utf-8");

    // Return context with error signal (orchestrator will handle exit 3)
    return {
      ok: false,
      exitCode: 3,  // Signal to orchestrator
      source: "validation",
      repoRoot: loaded.repoRoot,
      contractDir: null,
      cacheDir: null,
      warnings: [],
      files: {},
      summary: {},
      notes: [`Contract validation failed in COMPLIANCE mode (${validation.issues.length} issues)`]
    };
  }

  // Bind cache (warnings only, never blocks)
  const cache = bindCache({ repoRoot, contractHash: loaded.contractHash });

  // Build strategy context
  const strategy = buildStrategyContext({ loaded, cache });

  // Write runtime context artifact
  writeRuntimeContractContext({
    artifactsDir: args.artifactsDir,
    loaded,
    validation,
    cache,
    strategy
  });

  // Set environment variables (Phase 2.2 behavior)
  process.env.MINDTRACE_CONTRACT_DIR = loaded.contractDir;
  if (cache.valid) {
    process.env.MINDTRACE_PAGE_CACHE_DIR = loaded.cacheDir;
  }
  process.env.MINDTRACE_AUTOMATION_CONTRACT_CONTEXT_PATH =
    join(args.artifactsDir, "runtime-contract-context.json");

  // Return legacy format for backward compatibility
  return {
    ok: true,
    source: "artifacts",
    repoRoot: loaded.repoRoot,
    contractDir: loaded.contractDir,
    cacheDir: cache.valid ? loaded.cacheDir : null,
    warnings: validation.issues
      .filter(i => i.severity === "WARN")
      .map(i => `${i.code}: ${i.message}`),
    files: loaded.files,
    summary: strategy,
    notes: []
  };
}
```

## 4. Error Taxonomy & Enforcement

### 4.1 Issue Structure

```typescript
type ContractAwarenessIssue = {
  code: ContractAwarenessErrorCode;
  category: "contract" | "cache" | "schema" | "fingerprint";
  severity: "ERROR" | "WARN";
  message: string;
  details?: Record<string, unknown>;
};
```

### 4.2 Error Codes

```typescript
type ContractAwarenessErrorCode =
  // Contract errors (severity depends on mode)
  | "CA_CONTRACT_DIR_MISSING"   // Neither canonical nor legacy dir found
  | "CA_MISSING_FILE"            // Required contract file not found
  | "CA_REF_MISSING"             // Contract ref points to missing file
  | "CA_LEGACY_PATH"             // Using legacy .mindtrace/contracts/ (WARN)

  // Schema errors (always ERROR even in BEST_EFFORT)
  | "CA_SCHEMA_INVALID"          // AJV validation failed
  | "CA_JSON_PARSE_ERROR"        // JSON parse failure

  // Fingerprint errors (severity depends on mode)
  | "CA_HASH_MISSING"            // automation-contract.hash not found
  | "CA_HASH_MISMATCH"           // Stored hash != computed hash
  | "CA_HASH_COMPUTE_FAILED"     // Fingerprint computation failed

  // Version errors (severity depends on mode)
  | "CA_VERSION_UNSUPPORTED"     // schema_version N-2 or older
  | "CA_VERSION_OLD"             // schema_version N-1 (backward compatible)

  // Cache errors (always WARN)
  | "CA_CACHE_META_MISSING"      // Cache meta.json not found
  | "CA_CACHE_HASH_MISMATCH"     // Cache hash != contract hash
  | "CA_CACHE_READ_ERROR"        // Cache unreadable
  | "CA_CACHE_LEGACY_PATH";      // Using legacy cache structure
```

### 4.3 Severity Rules

```typescript
function getIssueSeverity(
  code: ContractAwarenessErrorCode,
  mode: "COMPLIANCE" | "BEST_EFFORT"
): "ERROR" | "WARN" {
  // Cache issues are ALWAYS warnings (never block execution)
  if (code.startsWith("CA_CACHE_")) return "WARN";

  // Catastrophic issues are always ERROR (even in BEST_EFFORT)
  if (code === "CA_SCHEMA_INVALID" || code === "CA_JSON_PARSE_ERROR") {
    return "ERROR";
  }

  // Other contract/fingerprint issues depend on mode
  if (mode === "COMPLIANCE") return "ERROR";
  return "WARN";  // Best effort downgrades to warnings
}
```

### 4.4 Mode Behavior Matrix

| Scenario | COMPLIANCE Mode | BEST_EFFORT Mode |
|----------|-----------------|------------------|
| Contract dir missing | ERROR → Exit 3 | WARN → Continue with degraded functionality |
| Contract file missing | ERROR → Exit 3 | WARN → Continue |
| Schema validation fails | ERROR → Exit 3 | ERROR (but no enforcement) |
| Hash mismatch | ERROR → Exit 3 | WARN → Continue |
| Ref points to missing file | ERROR → Exit 3 | WARN → Continue |
| Cache hash mismatch | WARN → Continue | WARN → Continue |
| Cache missing | WARN → Continue | WARN → Continue |
| Version N-1 | WARN → Continue | WARN → Continue |
| Version N-2 | ERROR → Exit 3 | WARN → Continue |

### 4.5 Exit Code Mapping

```typescript
function shouldExit3(issues: ContractAwarenessIssue[], mode: string): boolean {
  // Only in COMPLIANCE mode
  if (mode !== "COMPLIANCE") return false;

  // Any ERROR severity triggers exit 3
  return issues.some(i => i.severity === "ERROR");
}
```

**Exit Code Semantics:**
- **Exit 0** - Tests passed, policy satisfied, contract valid
- **Exit 1** - Tests failed (Playwright failures)
- **Exit 2** - Infrastructure/runtime error
- **Exit 3** - Compliance violation (contract invalid in COMPLIANCE mode)

## 5. Determinism & Authority Boundaries

### 5.1 Determinism Guarantees

**Byte-Identical Outputs:**
- Same repo + same contract files → same computed hash
- Same contract + same cache → same binding result
- Output artifacts are deterministically sorted (canonical JSON)

**No Timestamps:**
- `runtime-contract-context.json` excludes `generatedAt` field
- Use `schema_version` and `runId` for provenance instead

**Canonical JSON Sorting:**
```typescript
function canonicalStringify(obj: any): string {
  // Deep recursive key sorting at ALL depths
  return JSON.stringify(obj, (key, val) => {
    if (val && typeof val === "object" && !Array.isArray(val)) {
      return Object.keys(val)
        .sort()
        .reduce((sorted, k) => {
          sorted[k] = val[k];
          return sorted;
        }, {} as any);
    }
    return val;
  }, 2) + "\n";  // Trailing newline
}
```

**POSIX Path Normalization:**
- All paths stored as POSIX (forward slashes)
- Cross-platform consistency

### 5.2 Authority Boundaries (Strict)

**READ-ONLY Access:**
```typescript
// Allowed:
const contract = JSON.parse(readFileSync(contractPath, "utf-8"));
const cache = JSON.parse(readFileSync(cachePath, "utf-8"));

// FORBIDDEN:
writeFileSync(contractPath, ...);  // ❌ NEVER write to contract
writeFileSync(cachePath, ...);     // ❌ NEVER write to cache
```

**WRITE-ONLY Access:**
```typescript
// Allowed:
const artifactPath = join(artifactsDir, "phase2/runtime-contract-context.json");
writeFileSync(artifactPath, canonicalStringify(output), "utf-8");

// FORBIDDEN:
const badPath = join(repoRoot, ".mcp-contract/new-file.json");
writeFileSync(badPath, ...);  // ❌ NEVER write outside runs/
```

**No Side Effects:**
- No network calls
- No randomness (except UUID for temp files during atomic write)
- No environment mutations (except setting MINDTRACE_* vars for compat)
- No console output (write to artifacts instead)

## 6. Testing Strategy

### 6.1 Test Categories

**Unit Tests** (`__tests__/unit/`)

## 8. Acceptance Criteria

Phase 2.0 is "Done" when:

### 8.1 Determinism ✅
- ✅ Same repo + same contract → same computed hash
- ✅ Same contract + same cache → same `cache.valid` result
- ✅ Output `runtime-contract-context.json` is byte-identical across runs
- ✅ Canonical JSON sorting at all depths

### 8.2 Integrity ✅
- ✅ In COMPLIANCE mode: missing/invalid contract → Exit code 3 before tests
- ✅ In BEST_EFFORT mode: cache mismatch → WARN only, execution continues
- ✅ Catastrophic issues (JSON parse) → ERROR even in BEST_EFFORT

### 8.3 Authority Boundaries ✅
- ✅ Module never writes to `.mcp-contract/*` or `.mcp-cache/*`
- ✅ Module writes only under `runs/<runId>/artifacts/**`
- ✅ No `process.exit()` calls in library code

### 8.4 Observability ✅
- ✅ Always emits `runtime-contract-context.json` with:
  - Loaded contract hash
  - Schema versions
  - Cache binding status
  - Strategy context
  - Issues (code, severity, message)
- ✅ Writes `validation.json` when issues found

### 8.5 Backward Compatibility ✅
- ✅ Reads `.mcp-contract/` (canonical) first, then `.mindtrace/contracts/` (legacy)
- ✅ Reads `.mcp-cache/v1/` (canonical) first, then legacy `index.json`
- ✅ Phase 2.2 wrapper preserves existing return types
- ✅ Environment variables set for compatibility

### 8.6 Testing ✅
- ✅ 64+ tests passing (unit + integration + regression)
- ✅ 100% coverage of public API
- ✅ All critical scenarios covered (determinism, COMPLIANCE, dual-read, cache mismatch, schema compat)
